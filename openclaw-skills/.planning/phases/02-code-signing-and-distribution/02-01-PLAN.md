  <action>
Create adaptive Android launcher icons using the standard Android Studio template pattern. This is a minimal placeholder icon set — the OpenClaw claw visual identity will be applied post-beta.

1. Create the vector foreground drawable at `android/app/src/main/res/drawable/ic_launcher_foreground.xml`:
   - Use a simple vector: a rounded rectangle with the letter "C" (for Console), or a simple claw-silhouette path
   - Use `android:viewportWidth="108"`, `android:viewportHeight="108"` (standard adaptive icon canvas)
   - Color: `#FFFFFF` (white on colored background)

2. Create the background drawable at `android/app/src/main/res/drawable/ic_launcher_background.xml`:
   - Simple solid color: `#1A1A2E` (dark navy, matching the console aesthetic)

3. Create adaptive icon XML at `android/app/src/main/res/mipmap-anydpi-v26/ic_launcher.xml`:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <adaptive-icon xmlns:android="http://schemas.android.com/apk/res/android">
       <background android:drawable="@drawable/ic_launcher_background"/>
       <foreground android:drawable="@drawable/ic_launcher_foreground"/>
   </adaptive-icon>
   ```
   Copy same content to `ic_launcher_round.xml` in the same directory.

4. For legacy PNG densities (mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi), generate minimal 1x1 opaque PNG placeholders using Python's built-in `struct` and `zlib` modules (no external deps). The sizes are:
   - mdpi: 48x48
   - hdpi: 72x72
   - xhdpi: 96x96
   - xxhdpi: 144x144
   - xxxhdpi: 192x192

   Run this Python script to generate all PNGs:
   ```python
   import struct, zlib, os

   def make_png(w, h, color=(26, 26, 46, 255)):
       def chunk(name, data):
           c = zlib.crc32(name + data) & 0xffffffff
           return struct.pack('>I', len(data)) + name + data + struct.pack('>I', c)
       ihdr = struct.pack('>IIBBBBB', w, h, 8, 2, 0, 0, 0)
       r, g, b = color[:3]
       raw = b''.join(b'\x00' + bytes([r, g, b]) * w for _ in range(h))
       idat = zlib.compress(raw)
       return b'\x89PNG\r\n\x1a\n' + chunk(b'IHDR', ihdr) + chunk(b'IDAT', idat) + chunk(b'IEND', b'')

   densities = {'mdpi': 48, 'hdpi': 72, 'xhdpi': 96, 'xxhdpi': 144, 'xxxhdpi': 192}
   base = 'android/app/src/main/res'
   for density, size in densities.items():
       d = f'{base}/mipmap-{density}'
       os.makedirs(d, exist_ok=True)
       png = make_png(size, size)
       open(f'{d}/ic_launcher.png', 'wb').write(png)
       open(f'{d}/ic_launcher_round.png', 'wb').write(png)
   print('Done')
   ```
   Run from the project root: `python3 -c "$(cat above_script)"`

   Also create the anydpi-v26 directory: `mkdir -p android/app/src/main/res/mipmap-anydpi-v26`

  <name>Task 2: Add signingConfigs block to build.gradle.kts</name>
  <files>android/app/build.gradle.kts</files>
  <action>
Add a `signingConfigs` block to `android/app/build.gradle.kts` inside the `android {}` block, and wire it to the release buildType. Read env vars at build time — keystore credentials are never hardcoded.

Add after the `defaultConfig {}` block and before `buildTypes {}`:

```kotlin
signingConfigs {
    create("release") {
        val keystorePath = System.getenv("KEYSTORE_PATH")
        val keystorePassword = System.getenv("KEYSTORE_PASSWORD")
        val keyAlias = System.getenv("KEY_ALIAS")
        val keyPassword = System.getenv("KEY_PASSWORD")
        if (keystorePath != null && keystorePassword != null && keyAlias != null && keyPassword != null) {
            storeFile = file(keystorePath)
            storePassword = keystorePassword
            this.keyAlias = keyAlias
            this.keyPassword = keyPassword
        }
    }
}
```

Then update the `release` buildType to reference the signing config:

```kotlin
buildTypes {
    release {
        isMinifyEnabled = true
        proguardFiles(
            getDefaultProguardFile("proguard-android-optimize.txt"),
            "proguard-rules.pro"
        )
        val signingConfigRelease = signingConfigs.findByName("release")
        if (signingConfigRelease?.storeFile != null) {
            signingConfig = signingConfigRelease
        }
    }
}
```

Note: The `internal-distribution.yml` workflow already decodes the keystore to `/tmp/release.keystore` and passes `KEYSTORE_PATH=/tmp/release.keystore`, `KEYSTORE_PASSWORD`, `KEY_ALIAS`, `KEY_PASSWORD` as env vars to the `./gradlew assembleRelease` step.
