  <name>Task 1: Add apksigner verification step to Android distribution job</name>
  <files>.github/workflows/internal-distribution.yml</files>
  <action>
Add an `apksigner verify` step to the `android-firebase-internal` job in `internal-distribution.yml`, immediately after the "Build release APK" step and before the "Install Firebase CLI" step.

This step fails the job if the APK is unsigned — preventing silent distribution of unsigned builds (research pitfall pattern).

Insert after the `Build release APK` step:

```yaml
      - name: Verify APK is signed
        run: |
          set -euo pipefail
          APK_PATH="android/app/build/outputs/apk/release/app-release.apk"
          if [ ! -f "$APK_PATH" ]; then
            echo "❌ APK not found at $APK_PATH"
            exit 1
          fi
          # Use apksigner from Android SDK build-tools (available on ubuntu-latest via setup-java)
          APKSIGNER=$(find /usr/local/lib/android/sdk/build-tools -name "apksigner" -type f 2>/dev/null | sort -V | tail -1)
          if [ -z "$APKSIGNER" ]; then
            echo "⚠️ apksigner not found via SDK path, trying PATH"
            APKSIGNER=$(command -v apksigner || true)
          fi
          if [ -z "$APKSIGNER" ]; then
            echo "❌ apksigner not available — cannot verify APK signature"
            exit 1
          fi
          "$APKSIGNER" verify --verbose "$APK_PATH"
          echo "✅ APK signature verified"
```

Also verify the `assembleRelease` step already has all four signing env vars. Read the current workflow step. It should have:
```yaml
        env:
          KEYSTORE_PATH: /tmp/release.keystore
          KEYSTORE_PASSWORD: ${{ secrets.KEYSTORE_PASSWORD }}
          KEY_ALIAS: ${{ secrets.KEY_ALIAS }}
          KEY_PASSWORD: ${{ secrets.KEY_PASSWORD }}
```

If any env var is missing from the assembleRelease step, add it. The keystore is decoded to `/tmp/release.keystore` in the prior "Decode keystore" step — KEYSTORE_PATH must match this path.

Commit the change to a `feat/02-android-apksigner` branch, then open a PR to develop:
