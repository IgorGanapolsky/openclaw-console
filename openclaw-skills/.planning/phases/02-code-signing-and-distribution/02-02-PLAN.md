<objective>
Generate the Android release keystore (if not already existing), back it up to a password manager, and configure the four required GitHub Actions secrets so the internal-distribution.yml workflow can sign the release APK.

Purpose: A lost Android keystore is permanently unrecoverable — the Play Store treats a new keystore as a new app requiring a new bundle ID. This plan establishes the keystore before any CI signing work begins. Per research pitfall #4: "Store the .jks file in a password manager (1Password, Bitwarden) as a file attachment with the key alias and passwords before any CI work begins."

Output: Android release keystore backed up securely, four GitHub Actions secrets set in the production environment.
</objective>

<task type="auto">
  <name>Task 1: Check for existing keystore and generate if absent</name>
  <files></files>
  <action>
Check whether a keystore already exists. The ANDROID_KEYSTORE_BASE64 secret may already be set in GitHub if the user previously ran setup-secrets.sh.

Step 1 — Check GitHub secret existence:
```bash
gh secret list --repo IgorGanapolsky/openclaw-console --env production 2>/dev/null | grep -E "ANDROID_KEYSTORE_BASE64|KEYSTORE_PASSWORD|KEY_ALIAS|KEY_PASSWORD"
```

If all four secrets already exist, skip keystore generation and proceed to Task 2.

Step 2 — If ANDROID_KEYSTORE_BASE64 is missing, generate a new keystore:

IMPORTANT: Capture the generated passwords before running — use separate openssl calls and store them:
```bash
STORE_PASS=$(openssl rand -base64 32 | tr -dc 'a-zA-Z0-9' | head -c 32)
KEY_PASS=$(openssl rand -base64 32 | tr -dc 'a-zA-Z0-9' | head -c 32)
echo "Store password: $STORE_PASS"
echo "Key password:   $KEY_PASS"
keytool -genkeypair \
  -v \
  -keystore "$HOME/openclaw-release.jks" \
  -keyalg RSA \
  -keysize 4096 \
  -validity 10000 \
  -alias openclaw \
  -storepass "$STORE_PASS" \
  -keypass "$KEY_PASS" \
  -dname "CN=OpenClaw Console, OU=Mobile, O=OpenClaw, L=Remote, ST=CA, C=US"
```

Step 3 — Verify the keystore is valid:
```bash
keytool -list -keystore "$HOME/openclaw-release.jks" -storepass "$STORE_PASS" -alias openclaw
```
Output should show: `openclaw, PrivateKeyEntry, Certificate fingerprint (SHA-256): ...`

Step 4 — Set GitHub Actions secrets:
```bash
KS_BASE64=$(base64 -i "$HOME/openclaw-release.jks")
gh secret set ANDROID_KEYSTORE_BASE64 --body "$KS_BASE64" --repo IgorGanapolsky/openclaw-console --env production
gh secret set KEYSTORE_PASSWORD --body "$STORE_PASS" --repo IgorGanapolsky/openclaw-console --env production
gh secret set KEY_ALIAS --body "openclaw" --repo IgorGanapolsky/openclaw-console --env production
gh secret set KEY_PASSWORD --body "$KEY_PASS" --repo IgorGanapolsky/openclaw-console --env production
```

Step 5 — Verify all four secrets are now present:
```bash
gh secret list --repo IgorGanapolsky/openclaw-console --env production | grep -E "ANDROID_KEYSTORE_BASE64|KEYSTORE_PASSWORD|KEY_ALIAS|KEY_PASSWORD"
```
Should show all four names in the output.

5. Record all three pieces of information in the secure note:
