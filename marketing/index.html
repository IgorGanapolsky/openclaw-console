Quick Start Path

Up and running in under 10 minutes.

The fastest path to productive AI automation: install, connect your agent, install a skill, and start asking it to do real work on your machine.

1. Start the gateway

Run openclaw gateway start. The daemon starts and your agent gets all core tools: read, edit, exec, sessions, memory, web_search, image.

2. Connect your agent

Point Claude Code, Codex, or any MCP-compatible agent at the OpenClaw gateway. Core tools are available immediately — no configuration needed.

3. Install your first skill

Run clawhub install gh-issues or clawhub install 1password. Your agent can now triage GitHub issues, manage secrets, and more.

$ openclaw gateway start && clawhub install gh-issues && clawhub install 1password

How It Works

Three layers that make your agent truly capable

1

Core Tools

Every OpenClaw agent gets read, edit, exec, process, web_search, web_fetch, image, memory_search, and memory_get tools — always available, no configuration required.

2

Skills

Install skills from ClawHub or from the 50+ built-in skills. Each skill adds domain-specific capabilities: gh-issues for GitHub, 1password for secrets, coding-agent for spawning sub-agents, openhue for smart lights.

3

Memory & Sessions

MEMORY.md and memory/*.md persist context across sessions. The agent remembers your preferences, prior decisions, and project context. Sessions are durable and resumable with the process tool for long-running background work.

Core Tools

The tools that make everything possible

📖 read

Read file contents — text files and images (jpg, png, gif, webp). Supports offset/limit for large files. Output truncated to 2000 lines or 50KB per call.

✏️ edit

Make precise file edits using exact text replacement. Multiple targeted replacements per call. Creates parent directories automatically.

⚡ exec

Run shell commands with background continuation, PTY support for TTY-required CLIs, timeout, environment variables, and working directory control.

🔄 process

Manage running exec sessions: list, poll, log, write, send-keys, submit, paste, kill. Use for long-running background work, interactive CLIs, and coding agents.

🧠 memory

memory_search (semantic search across MEMORY.md + memory/*.md shards) and memory_get (exact excerpt reads). Persistent context across all sessions.

🌐 web_search + web_fetch

Search the web via Brave API. Fetch and extract readable content from URLs (HTML → markdown/text). Built-in research and documentation lookup.

Yes. OpenClaw Console works with Claude Code, OpenAI Codex, Gemini CLI, Amp, OpenCode, and any MCP-compatible agent. The coding-agent skill lets you spawn any of these agents as background subagents from within a conversation.
Local-first. OpenClaw Console runs entirely on your own machine. No cloud account is required. You own the data, the sessions, and the memory. Optional cloud features (like VPS node pairing via Tailscale) are available but not required for the core runtime.
The gh-issues skill lets your agent fetch GitHub issues, spawn sub-agents to implement fixes and open PRs, monitor PR review comments, and respond to reviews — all from a single command. Usage: /gh-issues [owner/repo] [--label bug] [--limit 5] [--milestone v1.0] [--assignee @me] [--fork user/repo] [--watch] [--interval 5] [--reviews-only] [--cron] [--dry-run] [--model glm-5] [--notify-channel -1002381931352].
The 1password skill sets up and uses the 1Password CLI (op). Use it to install the CLI, enable desktop app integration, sign in to single or multi-account setups, and read/inject/run secrets via op — all from your agent conversation.
OpenClaw uses a MEMORY.md file plus memory/*.md shards for persistent cross-session memory. The memory_search tool does semantic search across all memory files. The memory_get tool does exact excerpt reads. Agents automatically recall prior decisions, preferences, and project context across sessions.
Use the skill-creator skill: it guides you through creating, editing, auditing, and improving agent skills. You can also publish skills to ClawHub for others to install. Each skill is a directory with a SKILL.md file, optional scripts/ subdirectory, and optional references/ subdirectory. Trigger phrases like "create a skill", "tidy up a skill", "improve this skill", or "audit the skill" activate skill-creator automatically.
OpenClaw Console supports Codex as an agent runtime via the coding-agent skill. The Codex plugin install page (/codex-plugin.html) has the full setup guide for wiring Codex into an OpenClaw session as a spawnable sub-agent.
The OpenClaw gateway daemon is controlled via: openclaw gateway status, openclaw gateway start, openclaw gateway stop, and openclaw gateway restart. Run openclaw help for all available subcommands.
