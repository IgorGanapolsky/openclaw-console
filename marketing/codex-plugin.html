Codex Plugin Install Guide

How to wire OpenAI Codex into an OpenClaw Console session as a spawnable sub-agent runtime via the coding-agent skill.

1. Prerequisites

  • OpenClaw Console installed and gateway running (openclaw gateway start)
  • OpenAI Codex CLI installed and authenticated
  • Node.js >=18 on your machine

2. Install the coding-agent skill

The coding-agent skill handles spawning Codex (and Claude Code, Pi) as background sub-agents from within your OpenClaw session.

clawhub install coding-agent

Or use the built-in skill if you have a full openclaw-mono checkout — it's in skills/coding-agent/SKILL.md.

3. Spawn Codex from your agent

Once the skill is installed, ask your agent to delegate coding tasks to Codex. The coding-agent skill instructions specify: use PTY=true for Codex (required for interactive terminal UI).

# Example: ask OpenClaw to delegate to Codex "Use Codex to implement the user authentication feature in src/auth.ts" # The coding-agent skill will spawn Codex with pty:true # and manage the session via the process tool

4. Skill notes

  • Use Codex for: building/creating new features, reviewing PRs in temp dirs, refactoring large codebases, iterative coding that needs file exploration
  • NOT for: simple one-liner fixes (use edit tool directly), reading code (use read tool), or any work in the ~/clawd workspace
  • Codex requires pty:true — it's a TTY-required CLI
  • Claude Code: use --print --permission-mode bypassPermissions (no PTY needed)
  • Results auto-announce when done — do not busy-poll for completion
